# Glossary

Key terms used in ClawDevsAI documentation and codebase.

---

## A

**Agent**
An AI-powered role (dev_backend, QA, DevOps, etc.) that executes tasks. Each agent:
- Has own workspace with configuration (AGENTS.md, SOUL.md, etc.)
- Can access tools (shell, files, GitHub API, web search)
- Maintains persistent session history
- May have memory system enabled
- Runs on schedule (cron-based) or on-demand

See: [Agents Architecture](../architecture/agents.md)

**Agent Loop**
The execution cycle of an agent:
1. Receive message (user input or scheduled trigger)
2. Load workspace config and session history
3. Assemble system prompt
4. Call LLM for inference
5. Execute tools (if LLM decides to)
6. Stream results back
7. Persist to session history
8. Optional: update memory

**API Gateway**
See: OpenClaw Gateway

**AGENTS.md**
Configuration file in agent workspace. Defines:
- Agent mission and capabilities
- Tool usage rules
- Responsibility matrix
- Constraints and guardrails
- Integration with other agents

Example: `docker/base/openclaw-config/agents/dev_backend/AGENTS.md`

---

## B

**Bearer Token**
Authentication token for API requests. Format:
```
Authorization: Bearer <token>
```
In ClawDevsAI: `OPENCLAW_GATEWAY_TOKEN`

**Bootstrap**
Initial setup process:
1. Create workspace directories
2. Copy configuration files (from ConfigMaps)
3. Initialize credentials
4. Start agent service
5. Run health checks

**BOOTSTRAP.md**
One-time initialization file for new agent setup. Removed after first run (for CEO agent).

---

## C

**ConfigMap**
Docker Compose resource for non-secret configuration data. In ClawDevsAI:
- `openclaw-agent-config` — Agent specifications
- `openclaw-system-config` — System settings

**Control Panel**
Web UI for managing ClawDevsAI. Provides:
- Agent status monitoring
- Task creation and tracking
- Log viewing with filtering
- Real-time metrics

Stack: Next.js (frontend) + FastAPI (backend)

Port: 3000 (UI), 8000 (API)

**Cron Expression**
Schedule format for periodic tasks. Examples:
```
0 * * * *       # Every hour
0 */2 * * *     # Every 2 hours
0 9 * * 1       # Monday at 9 AM
0 0 1 * *       # First of month at midnight
```

Full reference: [crontab.guru](https://crontab.guru)

---

## D

**Docker**
Container runtime. Used for:
- Building images for Docker Compose containers
- Local development
- Reproducible environments

**DevOps**
Development Operations — infrastructure, deployment, monitoring

---

## E

**Embedding**
Vector representation of text. Used for:
- Memory search (semantic similarity)
- Finding relevant context
- Grouping similar concepts

In ClawDevsAI: Generated by Ollama (nomic-embed-text) or cloud providers

---

## F

**FastAPI**
Python web framework for building APIs. Used for Control Panel backend.
- Async/await support
- Automatic OpenAPI documentation
- Type hints validation

---

## G

**Gateway**
See: OpenClaw Gateway

**GPU**
Graphics Processing Unit. Optional hardware acceleration for:
- LLM inference (faster token generation)
- Embedding calculation
- In ClawDevsAI: NVIDIA GPUs via Device Plugin

**Git Hooks**
Scripts that run on git events (commit, push). In ClawDevsAI:
- Prevent commits to main/master
- Format validation
- Security checks

---

## H

**Heartbeat**
Lightweight periodic execution. Uses smaller context than full agent loop to save tokens.
- Limited to essential tasks
- Faster execution
- Lower cost

See: OpenClaw documentation on Heartbeat vs Cron

---

## I

**Identity.md**
Agent configuration file defining:
- Name and role
- Emoji and visual identity
- Immutable constraints (anti-prompt-injection)
- Key characteristics

---

## K

**Docker Compose (container)**
Container orchestration platform. Provides:
- Container scheduling
- Service discovery
- Persistent storage
- Secrets management

In ClawDevsAI: Docker for local development, multi-node cluster for production

**Kustomize**
Docker Compose templating tool. Enables:
- Overlays for different environments
- Reusable base configurations
- ConfigMap/Secret generation

Location: `docker/base/` (base) and `docker/overlays/` (variations)

---

## L

**LLM**
Large Language Model (e.g., GPT-4, Claude, Nemotron). In ClawDevsAI:
- Ollama (local)
- Anthropic/OpenAI (cloud, via OpenRouter)

**Log Level**
Severity filter for logging:
- `debug` — Detailed function entry/exit, state changes
- `info` — Important events (agent started, tool called)
- `warn` — Unexpected behavior (fallback, retry)
- `error` — Failures (exception, abort, timeout)

---

## M

**Memory (Agent Memory)**
Persistent learning system for agents:
- **MEMORY.md** — Curated long-term facts
- **memory/YYYY-MM-DD.md** — Daily logs
- **Embeddings** — Vector search for semantic recall
- **TTL** — Time-to-live (aging old memories)

See: [Memory System](../architecture/memory-system.md)

**Docker**
Local Docker Compose cluster for development. Single-node, easy setup.

```bash
make docker-up    # Start
make docker-down  # Stop
```

---

## O

**Ollama**
Local LLM runtime. Provides:
- Model serving (nemotron, qwen, etc.)
- Embedding generation
- REST API on port 11434

Models run locally without cloud dependency.

**OpenClaw**
Agent orchestration framework. Manages:
- Sessions and history
- Tool execution
- Multi-channel integration (Telegram, Discord, WebSocket)
- Authentication and authorization

See: [OpenClaw Documentation](https://docs.openclaw.ai)

**OpenClaw Gateway**
Central component of OpenClaw. Handles:
- JSON-RPC requests from clients
- Session persistence
- Tool routing
- WebSocket connections

Port: 8080 (internal), 18789 (exposed)

---

## P

**PVC (PersistentVolumeClaim)**
Docker Compose storage request. In ClawDevsAI:
- `ollama-data` — 200 GB for models
- `openclaw-data` — 100 GB for sessions/logs
- `panel-db` — 20 GB for metadata

**Container**
Smallest deployable unit in Docker Compose. Contains one or more containers.

In ClawDevsAI:
- `openclaw-runtime` — Agent orchestration
- `ollama-runtime` — LLM server
- `clawdevs-panel-backend` — API
- `clawdevs-panel-frontend` — Web UI
- `postgres` — Database

---

## Q

**Query**
User input or message sent to an agent. Triggers:
- Agent execution
- LLM inference
- Tool selection
- Response generation

---

## R

**Role Prompting**
Technique where model is given a specific role (architect, QA, etc.) to improve response quality.

Example: "Acting as a security engineer, review this code for vulnerabilities."

**RPC (Remote Procedure Call)**
Protocol for calling functions across network boundaries. In ClawDevsAI:
- JSON-RPC over WebSocket
- Async request/response

---

## S

**SDD (Structured Development Development)**
Workflow template for organized development:

```
CONSTITUTION → BRIEF → SPEC → CLARIFY → PLAN → TASK → IMPLEMENTATION
```

Templates in: `docker/base/openclaw-config/shared/`

**Secret**
Docker Compose resource for sensitive data:
- API tokens
- Database passwords
- OAuth credentials

Never committed to git. Generated from `container/.env`.

**Service**
Docker Compose abstraction for accessing containers. Provides:
- Stable IP address
- Load balancing
- DNS name

Types: ClusterIP (internal), NodePort (external), LoadBalancer

**Session**
Persistent conversation between user and agent:
- Stores all messages
- Maintains context
- Reused across executions
- Can be pruned (old messages removed)

Storage: RocksDB (session files) + PostgreSQL (metadata)

**SOUL.md**
Agent configuration file defining:
- Values and principles
- Non-negotiable constraints (hard guardrails)
- Tone and communication style
- Behavioral guidelines

---

## T

**Telegram**
Messaging platform for agent notifications and interaction:
- Bot receives messages
- Agent responds
- Notifications sent to CEO

Configuration: `TELEGRAM_BOT_TOKEN_CEO`, `TELEGRAM_CHAT_ID_CEO`

**Tool**
Callable function available to agents:
- Shell commands (`sh`, `bash`)
- File operations (`read`, `write`, `edit`)
- API calls (GitHub, web search)
- Code execution (Python, JavaScript)

Tools are sandboxed and permission-controlled.

---

## U

**USER.md**
Agent configuration file containing:
- Primary user identity
- Escalation contacts
- Timezone
- Preferences and context

---

## V

**Vector**
Numerical representation of text/embeddings. Used for:
- Semantic similarity search
- Memory retrieval
- Clustering related concepts

**Vector Database**
Specialized database for embedding storage and search:
- Similarity queries
- Approximate nearest neighbor
- In ClawDevsAI: SQLite + vector extension (or cloud)

---

## W

**WebSocket**
Protocol for real-time bidirectional communication. In ClawDevsAI:
- Client ↔ OpenClaw Gateway
- Streaming responses
- Event subscriptions

**Workspace**
Agent's persistent directory. Contains:
- Configuration files (AGENTS.md, SOUL.md, etc.)
- Memory files
- Session history
- Temporary data

Location: `/data/openclaw/workspace-<agent-id>/`

---

## X-Z

**YAML**
Configuration file format used for:
- Docker Compose manifests
- Kustomize overlays
- Configuration data
