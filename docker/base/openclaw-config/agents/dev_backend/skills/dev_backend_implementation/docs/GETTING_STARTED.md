# Getting Started

## When to Use This Skill

- Scheduled queue cycle for backend issues
- Immediate handoff from Architect
- Before writing production code (for requirements analysis)
- After implementation (for evidence validation)

## Quick Steps

1. **Analyze Requirements** → Hook: `before_execution`
   - Conversation context
   - Project config
   - System prompt

2. **Get Recommendation** → Architecture matrix
   - Tech stack suggestion
   - Pattern recommendation
   - Trade-off analysis

3. **Implement** → Following recommended stack
   - Write code
   - Add tests (>=80% coverage)
   - Run quality gates

4. **Report** → Hook: `after_execution`
   - Metrics and logs
   - Evidence report
   - Cost/performance trade-offs
