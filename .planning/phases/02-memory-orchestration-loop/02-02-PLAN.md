  <action>
    Create `control-panel/backend/app/services/parallelism_gate.py` with:
    - `PARALLELISM_EVENT_TYPE = "task.parallelism_blocked"`.
    - `THRESHOLD_EVENT_TYPE = "task.parallelism_thresholds_updated"`.
    - `async def fetch_parallelism_metrics(session, lookback: int) -> dict` that:
      - Selects last `lookback` tasks with `status == "done"` ordered by `updated_at` desc.
      - Computes `avg_cost` from `Task.actual_cost` (0 if None).
      - Computes `avg_latency_seconds` from `(updated_at - created_at).total_seconds()` when both present.
      - Computes `p95_cost` and `p95_latency_seconds` from the same samples.
      - Returns `{ "sample_size": int, "avg_cost": float, "avg_latency_seconds": float, "p95_cost": float, "p95_latency_seconds": float }`.
    - `def compute_adaptive_thresholds(metrics: dict, settings) -> dict` that:
      - Requires `metrics["sample_size"] >= settings.ORCH_PARALLELISM_ADAPTIVE_MIN_SAMPLES`; otherwise return base thresholds.
      - Computes `adaptive_cost = max(settings.ORCH_PARALLELISM_COST_THRESHOLD, metrics["avg_cost"] * settings.ORCH_PARALLELISM_COST_MULTIPLIER, metrics["p95_cost"])`.
      - Computes `adaptive_latency = max(settings.ORCH_PARALLELISM_LATENCY_THRESHOLD_SECONDS, metrics["avg_latency_seconds"] * settings.ORCH_PARALLELISM_LATENCY_MULTIPLIER, metrics["p95_latency_seconds"])`.
      - Returns `{ "cost_threshold": float, "latency_threshold_seconds": float }`.
    - `async def persist_adaptive_thresholds(session, thresholds: dict) -> None` that:
      - Upserts a `SystemSetting` (or existing key-value config model) for `orch.parallelism.cost_threshold` and `orch.parallelism.latency_threshold_seconds`.
      - Uses `log_task_event` to record `THRESHOLD_EVENT_TYPE` with payload `{ "cost_threshold": ..., "latency_threshold_seconds": ..., "sample_size": ... }`.
    - `def should_allow_parallelism(in_progress_count: int, metrics: dict, settings) -> bool` that:
      - If `settings.ORCH_PARALLELISM_FORCE` is True → return True.
      - If `settings.ORCH_PARALLELISM_ENABLED` is False → return False.
      - If `in_progress_count == 0` → return True.
      - Otherwise allow only if `metrics["avg_cost"] <= settings.ORCH_PARALLELISM_COST_THRESHOLD`
        AND `metrics["avg_latency_seconds"] <= settings.ORCH_PARALLELISM_LATENCY_THRESHOLD_SECONDS`.
    - `async def evaluate_parallelism_gate(session, in_progress_count: int) -> tuple[bool, str]` that:
      - Loads metrics via `fetch_parallelism_metrics` with `settings.ORCH_PARALLELISM_LOOKBACK_TASKS`.
      - Computes adaptive thresholds via `compute_adaptive_thresholds` and updates settings in-memory for evaluation.
      - Persists adaptive thresholds with `persist_adaptive_thresholds` when `settings.ORCH_PARALLELISM_ADAPTIVE_ENABLED` is True.
      - Returns `(allowed, reason)` where reason is one of: `"force_enabled"`, `"disabled"`, `"sequential_allowed"`, `"thresholds_met"`, `"thresholds_exceeded"`.

    Update `control-panel/backend/app/core/config.py` to add settings:
    - `ORCH_PARALLELISM_ENABLED: bool = Field(default=False)`
    - `ORCH_PARALLELISM_FORCE: bool = Field(default=False)`
    - `ORCH_PARALLELISM_COST_THRESHOLD: float = Field(default=2.0)`
    - `ORCH_PARALLELISM_LATENCY_THRESHOLD_SECONDS: int = Field(default=600)`
    - `ORCH_PARALLELISM_LOOKBACK_TASKS: int = Field(default=25)`
    - `ORCH_PARALLELISM_ADAPTIVE_ENABLED: bool = Field(default=True)`
    - `ORCH_PARALLELISM_ADAPTIVE_MIN_SAMPLES: int = Field(default=10)`
    - `ORCH_PARALLELISM_COST_MULTIPLIER: float = Field(default=1.2)`
    - `ORCH_PARALLELISM_LATENCY_MULTIPLIER: float = Field(default=1.2)`

    Add `control-panel/backend/tests/test_parallelism_gate.py` tests:
    - Disabled gate returns False.
    - Force returns True.
    - Thresholds met returns True when in_progress_count > 0.
    - Adaptive thresholds use rolling metrics to raise thresholds and are persisted.
