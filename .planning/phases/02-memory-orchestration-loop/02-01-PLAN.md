      - Validates using `validate_contract(contract_name, parsed)`; on invalid, raise `ValueError` with joined error messages.

      1. Set `workflow_state=WORKFLOW_PLANNING`, log `task.planned` with payload `{contract: "plan"}`.
      2. Set `workflow_state=WORKFLOW_EXECUTING`, log `task.executed` with payload `{contract: "execute"}`.
      3. Set `workflow_state=WORKFLOW_SELF_REVIEW`, log `task.self_reviewed` with payload `{contract: "self_review"}`.
      4. Set `workflow_state=WORKFLOW_PEER_REVIEW`, log `task.peer_reviewed` with payload `{contract: "peer_review"}` using agent_slug `"ceo"` as reviewer.
      5. Set `workflow_state=WORKFLOW_CONSOLIDATING`, log `task.consolidated` with payload `{contract: "consolidate"}`.
      6. Set `task.status="done"` and `workflow_state=WORKFLOW_COMPLETED` and commit.
    - Each step must build prompts that demand JSON output with the exact contract fields:
      - Plan prompt includes: task id/title/description and the instruction `"Return JSON only with keys: task_id, task_title, task_description, plan_steps, risk_notes."`
      - Execute prompt includes: task id + plan_steps and the instruction `"Return JSON only with keys: task_id, plan_steps, actions, artifacts, evidence, errors."`
      - Self-review prompt includes: task id + executed actions/evidence and `"Return JSON only with keys: task_id, checks, issues, decision."`
      - Peer-review prompt includes: task id + self-review output and `"Return JSON only with keys: task_id, reviewer, issues, decision."`
      - Consolidate prompt includes: task id + peer review output and `"Return JSON only with keys: task_id, summary, artifacts, evidence, next_steps."`
    - On any validation or execution exception during plan/execute/review steps:
      - Increment `task.workflow_attempts` and if attempts < MAX_WORKFLOW_ATTEMPTS, log `task.replan_requested` and restart from planning.
      - If attempts >= MAX_WORKFLOW_ATTEMPTS, set `workflow_state=WORKFLOW_FAILED`, set `workflow_last_error`, log `task.failed`.
    - Preserve idempotency: if task is already in `FINAL_WORKFLOW_STATES`, return early.
    - Ensure `log_task_event` is called for every step with `event_type` values above and payload includes `from_agent_slug` and `to_agent_slug` where applicable.
