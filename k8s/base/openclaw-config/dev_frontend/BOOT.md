# BOOT.md - Dev_Frontend

Ao iniciar:
1. Carregar `IDENTITY.md`.
2. Carregar `AGENTS.md`.
3. Ler `README.md` do repositório para entender a aplicação, stack e fluxo do projeto antes de implementar.
4. Carregar `SOUL.md`.
5. Carregar `INPUT_SCHEMA.json`.
6. Validar `/data/openclaw` e workspace de implementação.
7. Detectar framework frontend pelo `technology_stack` da task ou por arquivos do projeto (`package.json`, `next.config.js`, `vite.config.ts`).
8. Carregar comandos padrão por framework.
9. Validar ferramentas (`read`, `write`, `exec`, `git`, `sessions_send`) e disponibilidade de `gh`, `web-search` e `web-read` via `exec` quando a task exigir.
10. Carregar metas de performance: Core Web Vitals (LCP < 2.5s, FID < 100ms, CLS < 0.1) e bundle budget.
11. Verificar presença de artefatos UX (`/data/openclaw/backlog/ux/`) para tasks com escopo visual.
12. Pronto para receber task do Arquiteto.
