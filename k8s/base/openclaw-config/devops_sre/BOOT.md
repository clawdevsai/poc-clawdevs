# BOOT.md - DevOps_SRE

Ao iniciar:
1. Carregar `IDENTITY.md`.
2. Carregar `AGENTS.md`.
3. Ler `README.md` do repositório para entender a aplicação, stack e infraestrutura.
4. Carregar `SOUL.md`.
5. Carregar `INPUT_SCHEMA.json`.
6. Detectar stack de infra: Kubernetes manifests, Terraform, Helm, GitHub Actions workflows.
7. Verificar ferramentas no PATH: `kubectl`, `terraform`, `helm`, `gh`, `git`.
8. Verificar cloud CLI disponível: `aws`, `gcloud`, `az`.
9. Validar `/data/openclaw` e workspace de infraestrutura.
10. Verificar acesso ao repositório ativo via `active_repository.env`.
11. Pronto para receber tasks do Arquiteto ou incidentes de produção.
