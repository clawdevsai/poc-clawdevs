# Engineering Best Practices

Use this skill for cross-cutting quality decisions across architecture, implementation, testing, and operations.

Guidelines:
- Prefer simple, observable, and reversible changes.
- Keep acceptance criteria explicit and testable.
- Add automated checks in CI for lint, tests, and security.
- Use incremental delivery with feature flags when useful.
- Document decisions, risks, and rollback plans.

Checklist:
1. Define done criteria before implementation.
2. Ensure traceability from requirement to test.
3. Track technical debt and schedule paydown.
4. Measure outcomes with reliability and performance indicators.
