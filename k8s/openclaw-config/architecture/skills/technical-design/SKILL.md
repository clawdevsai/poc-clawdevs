# Technical Design

Use this skill when the PO asks for stack definition, architecture decisions, or detailed implementation tasks.

Workflow:
1. Read the approved idea and the relevant user story files in `/data/openclaw/backlog`.
2. Research the most relevant best practices and technology options on the internet.
3. Choose the stack and architecture with explicit tradeoffs, prioritizing low cost and high performance.
4. Generate one or more task files per user story in `/data/openclaw/backlog/tasks`.
5. Make each task directly executable by an engineering team with scope, acceptance criteria, dependencies, and suggested tests.
