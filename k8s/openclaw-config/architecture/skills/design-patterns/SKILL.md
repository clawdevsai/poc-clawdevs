# Design Patterns

Use this skill when selecting reusable design approaches to solve recurring software problems.

Guidelines:
- Choose patterns only when they reduce real complexity.
- Prefer simple composition before deep inheritance.
- Document why a pattern is used and its tradeoffs.
- Avoid pattern stacking that harms readability.
- Revisit pattern choice as requirements evolve.

Checklist:
1. Describe the concrete problem first.
2. Compare at least one simpler alternative.
3. Pick the pattern with lowest operational cost.
4. Add tests that validate expected flexibility.
