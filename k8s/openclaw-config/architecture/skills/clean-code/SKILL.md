# Clean Code

Use this skill when writing or reviewing implementation details for clarity and long-term maintenance.

Guidelines:
- Prefer clear names over comments that explain confusion.
- Keep functions small and focused on one intent.
- Remove dead code and hidden side effects.
- Keep error handling explicit and predictable.
- Refactor in small safe steps with tests.

Checklist:
1. Validate naming, cohesion, and readability.
2. Split long methods and large files when needed.
3. Replace magic values with named constants.
4. Ensure tests cover behavior, not internals.
