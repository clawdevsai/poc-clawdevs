This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**ClawDevs AI** is a full-stack AI agent orchestration platform. It runs a local stack of 10 Docker containers: a Next.js control panel, a FastAPI backend, an OpenClaw agent gateway, Ollama for local LLM inference, PostgreSQL (with pgvector for RAG), Redis (queue + cache), SearXNG (web search), and supporting services.

The Makefile is the primary way to operate the stack. There is no docker-compose.yml.

## Essential Commands

### Stack Operations
```bash
make up-all           # Build + start all 10 containers (first run)
make up-all-with-cache # Start all with Docker layer cache
make down             # Stop all containers
make restart          # Stop + restart
make status           # Container status
make logs             # Aggregated logs
make logs-follow      # Real-time logs
make env-check        # Validate .env before starting
```

### Individual Services
```bash
make up-<service>     # e.g., make up-postgres, make up-openclaw
make openclaw-restart # Restart OpenClaw agent gateway (no rebuild)
```

### Database
```bash
make panel-db-migrate # Run Alembic migrations
```

### Cleanup
```bash
make reset            # Remove containers + volumes
make destroy          # Remove containers, images, cache
```

### Frontend Development
```bash
cd control-panel/frontend
pnpm install
pnpm dev              # Next.js with Turbopack at localhost:3000
pnpm build            # Production build
pnpm lint             # TypeScript type check (tsc --noEmit)
pnpm orval            # Regenerate API clients from OpenAPI spec
```

### Backend Development
```bash
cd control-panel/backend
uv sync               # Install Python deps
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### Tests
```bash
cd control-panel/backend
uv run pytest tests/            # All tests
uv run pytest tests/test_foo.py # Single test file
```

## Architecture

### Request Flow
1. Browser → Next.js frontend (`:3000`)
2. Next.js rewrites `/api/*` → FastAPI backend (`:8000`)
3. React Query clients (auto-generated by Orval from `/openapi.json`) call backend
4. Long-running work → Redis → RQ workers
5. Real-time updates pushed via WebSocket (`/ws`)

### Backend Structure (`control-panel/backend/app/`)
- `main.py` — FastAPI app with lifespan (runs migrations, bootstraps admin, syncs agents, starts health monitor)
- `routers/` — Route handlers (auth, agents, sessions, tasks, approvals, memory, crons, metrics, cluster, etc.)
- `models/` — SQLModel ORM models (maps to PostgreSQL tables)
- `services/` — Business logic (agent sync, RAG, OpenClaw client, governance, circuit breaker, cost tracker)
- `tasks/` — RQ async tasks (periodic sync, task orchestration)
- `core/` — Config (pydantic-settings), database session factory, auth utilities
- `hooks/` — OpenClaw tool execution callbacks

### Frontend Structure (`control-panel/frontend/`)
- App Router (Next.js 16) with Turbopack
- `src/lib/api/` — Orval-generated React Query + Axios clients
- `src/lib/ws.ts` — WebSocket client
- `next.config.ts` — Rewrites `/api/*` and `/api/ws/*` to the backend

### Data Layer
- **PostgreSQL 15** with **pgvector** — primary store + vector embeddings for RAG
- **Redis** — RQ task queue and caching
- **Alembic** — migrations (auto-run at backend startup)

### Agent Communication
- **OpenClaw Gateway** (`:18789`) — receives agent tool calls, executes them, returns results
- `services/openclaw_client.py` — backend HTTP client to the gateway, authenticated with `OPENCLAW_GATEWAY_TOKEN`
- Agents run on configurable cron schedules (env vars like `DEV_BACKEND_CRON_EXPR`)

### LLM Providers
- **Ollama** (local, `:11434`) — default
- **OpenRouter** (cloud) — switch via `PROVEDOR_LLM=openrouter` in `.env`

## Environment Setup

Copy `.env.example` to `.env` and configure:
- `OPENCLAW_GATEWAY_TOKEN` — shared internal auth token
- `PANEL_SECRET_KEY` — JWT secret (run `make gen-secret`)
- `PANEL_ADMIN_USERNAME` / `PANEL_ADMIN_PASSWORD` — control panel login
- `PANEL_DB_PASSWORD` / `PANEL_REDIS_PASSWORD` — service passwords
- `GIT_TOKEN` / `GIT_ORG` — GitHub integration
- `TELEGRAM_BOT_TOKEN_CEO` / `TELEGRAM_CHAT_ID_CEO` — notifications
- `PROVEDOR_LLM` — `ollama` or `openrouter`

## Key Conventions

- **Frontend package manager**: pnpm — always use `pnpm`, never `npm` or `yarn`
- **Backend package manager**: uv — always use `uv run` / `uv sync`, never `pip`
- **API client generation**: After changing backend schemas, run `pnpm orval` in the frontend to regenerate TypeScript clients
- **Migrations**: Model changes require a new Alembic migration; they run automatically on backend startup
- **Documentation language**: README and docs are in Portuguese (pt-BR); code is in English
